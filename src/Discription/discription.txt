1. Весь код разнесен по слоям: Infrustructure, Application, Domain
2. Из EmployeeKpiController вынесена бизнес-логика: обращения к репозиториям
3. Из экшенов EmployeeKpiController вынесены зависимости: все new Service из тела экшенов вынесены в параметры
   для исключения жесткой зависимости, фреймворк сам создаст зависимости при обращении к контроллеру
4. Из консольных команд вынесены зависимости: все new Service из тела execute() вынесены в конструктор для исключения
   жесткой зависимости, фреймворк сам создаст зависимости при вызове команды
5. Для консольных команд создан абстрактный класс для масштабирования кода в будущем (Open-Closed)
6. Для сервисов созданы интерфейсы для масштабирования кода в будущем (Open-Closed)
7. Для полей сущностей созданы Value Objects для валидации охранными методами и исключения возможности перепутать
   однотипные поля между собой
8. Соблюдена инверсия зависимостей: зависимости направлены внутрь, модули зависят от абстракций
